# git_demo
This is a description

Few GIT commands:

# GENERAL
git --version
pwd
ls
git cd
git status
cd .. --------> Used to go back a folder
mkdir   -----------> To make new directory
git init    ------> To make it git repository
touch .gitignore

# CONFIG
git config --global user.name "My name"
git config --global user.email "Email"
git config --global core.editor "code --wait"
git config --list

# LOG, ADD, COMMIT, PUSH
git log --oneline     ----> Gives results in one line
git add *file name*
git add .
git commit -m "Message"
git push origin main
git push -u origin main  ----------> Can use git push next time directly

# REMOTE ------ To connect to an online GITHUB repo
git remote add origin <link>
git remote -v    -----> To verify remote directory

# CLONE -------- To directly start working on an existing GITHUB Repo
git clone link

# FETCH ---------> Gets info from online repo but doesn't make changes.
git fetch

# PULL ----------> Gets the info and adds to my work. git pull = git fetch + git merge
git pull origin main ----> Changes will be merged to main

# BRANCH
git branch    -----> To check branch
git branch -M (new_name)    -----> To rename the branch
git switch new_branch_name
git checkout branch_name    -------> To navigate to another branch
git checkout -b new_branch_name    ----> To create a new branch
git switch -c new_branch_name    ----> To create a new branch and move there
git branch -d branch_name   -----> To delete branch

# Merge Code ( Two ways: Using Pull Request or use Terminal)
git checkout master  -------- Make sure you go to the master branch first
git merge branch_name    -----------> To merge 2 branches into master branch

# DIFF
git diff branch_name    -----------> To compare commits, branches, files & more
git diff --staged ----------> Shows the changes to files in the staging area.
git diff log_code log_code_newer ----> Shows changes between a file in these two logs
git diff log_code..log_code_newer ----> Shows changes between a file in these two logs
git diff branchone..branchtwo

# GIT STASH ---- used to switch branches before commiting or adding changes to the current branch.
git stash ------- to stash or temporarily save the changes
git stash pop -------- to get them back after you go back to the branch you were working on

# PULL
git pull origin main

# UNDOING CHANGES
Case 1: Staged changes: (Only added)
git reset <-file name->
git reset -----------> All files reset at once
Case 2: Commited Changes: (For one commit)
git reset HEAD~1
Case 3: Commited Changes: (For many commits)
git log ------> Get the hash/code of the change we want to go back to and replace that with hash below

# RESET
git reset <-commit hash->
git reset --hard <-commit hash->

# RESTORE
git restore filename ---> To restore a previous version of file.
git reflog -----> To go back to previous state

# REBASE  ------- never use from main/master branch. It
git rebase master  -----------> means i am merging my current branch with master
